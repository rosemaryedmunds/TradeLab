## What this is

A single-process Node app that stores options trades in SQLite and renders three classes of pages:
- **Legacy analytical dashboards** (`/`, `/today/`, `/csv/`) — verbatim HTML/JS copied from a previous static-file site. The server injects fresh trade data into a `<script id="trade-data" type="application/json">…</script>` block at request time so the client JS runs unchanged. **Don't refactor the dashboard JS unless asked** — it works and the user is attached to its look/feel.
- **Manage page** (`/trades`, `/manage`) — modern CRUD with CSV upload, written from scratch.
- **Per-trade chart** (`/charts/spx-trade/?date=YYYY-MM-DD&...`) — lightweight-charts SPX candles with trade markers, indicators (Pivot Ribbon EMAs, Phase Oscillator, ATR levels).

The DB at `data/tradelab.db` is the single source of truth. Every other view derives from it.

## Critical invariants (don't break these)

1. **The legacy HTML files in `public/{index,today/index,csv/index}.html` must keep their existing DOM structure and the `<script id="trade-data" type="application/json">…</script>` block.** The on-disk content of the block is `[]` (scrubbed of user data) and `template.js` substitutes the real trade JSON at request time. If you change the marker tag's attributes or remove the script element, the dashboards go blank.

2. **The legacy "baked JSON shape" must be preserved exactly.** Fields the dashboards read: `id, seq, symbol, root, expiry, strike, right, direction, quantity, entry_dt, exit_dt, entry_date, exit_date, entry_time, exit_time, entry_price, exit_price, entry_gross_cost, exit_gross_proceeds, gross_pnl, net_pnl, pnl_pct, duration_min, fee_drag, costs, total_commission, synthetic_exit, source_rows, execution_ladder, cum_pnl, drawdown, round_trip_id`. The shape is generated by `server/legacyShape.js`. **Datetime fields are space-separated** (`"2026-01-06 11:57:58"`), not ISO with `T`, because the legacy JS does direct string ops.

3. **Trades with `exit_dt: NULL` are filtered out of legacy dashboards.** `toMs(null)` returns `NaN`, which breaks every Math.min/max and date sort downstream. Always run new open trades through the auto-expiry path or leave them excluded.

3a. **Timezone of `entry_dt` / `exit_dt` is empirically the user's local wall-clock, not ET.** IBKR Flex Queries and Webull both export in the user-configured local TZ; that's what the importer accepts and writes verbatim. The legacy dashboards (`/`, `/today/`, `/csv/`) still treat the stored string as ET via `parseSourceMs()` (SOURCE_TZ = `America/New_York`) and convert to CT — which produces a 1-hour shift relative to reality for a CT-based account. **The Trade Doctor (`server/doctor.js`) treats stored times as wall-clock in the user's TZ** (constant `STORED_TZ_OFFSET_TO_BUCKET_HOURS = 0`). If you later need true-ET handling (a New York account importing under a NY-configured broker), the right fix is a per-trade `source_tz` column at import time, not a global flip. Legacy dashboards should eventually be reconciled to this convention; for now they remain on the old contract per "don't refactor legacy unless asked."

4. **CSV dedup key = `(symbol, entry_dt, exit_dt, exit_price)`.** A single buy that's FIFO-split into N closing sells produces N rows with the same `entry_dt` but different `exit_dt`/`exit_price` — they must remain distinct. Don't simplify back to `(symbol, entry_dt)` (that bug cost ~$150 in lost trades during initial development).

5. **In tape-format imports, sort fills by `(dt asc, BUY before SELL)`.** Broker tapes round to the minute, and the file is often reverse-chronological. Without the BUY-first tiebreak, FIFO sees a SELL with no open position and drops the round trip.

6. **Option multiplier = 100.** No exceptions in the current dataset (all SPX/SPXW/SPY/XSP/AVGO options). If you add futures or stock, branch on `t.root` or add a multiplier column.

7. **Commission sign convention = negative.** Stored as a cost. Broker CSVs typically have positive values; `csvImport.js` flips them.

## Indicator math — sourced from `/root/spy/indicators.py` (off-repo)

`server/indicators.js` is a 1:1 JS port of the original Python:
- `applyPivotRibbon`: EMA(8/13/21/48/200), fast/slow cloud bullish flags, BB compression vs ATR threshold, candle bias
- `applyPhaseOscillator`: `((close - EMA(close,21)) / (3 * ATR14)) * 100`, then EMA(3); zones at ±23.6 / ±61.8 / ±100
- `applyAtrLevels`: daily-anchored, **prior-period** close + prior-period ATR-14 (Saty `period_index=1`)

If you tune any of these, mirror the change in the Python source comment-wise so they stay in sync.

## Massive.com API quirks

- Use `query2.finance.yahoo.com` style endpoint at `api.massive.com` — Polygon-compatible API surface
- `I:SPX` is the cash index ticker (use this, not `SPX` or `^SPX`)
- **Don't use a "real browser" User-Agent.** Yahoo (which this proxies) detects them as bots. Use the bare string `Mozilla/5.0`.
- **Spawn curl as a subprocess** (`execFile`); Node's native `fetch` has a TLS fingerprint Massive blocks for SPX intraday.
- Cache every successful response — `data/candle-cache/SPX_<date>_1m.json` and `SPX_<date>_1m_daily.json`. Today's session is never cached.

## Settlement logic for expired options

`server/settlement.js` computes intrinsic value at PM settlement (4:00 PM ET cash close):
- Calls: `max(0, S - K)`
- Puts: `max(0, K - S)`

For AM-settled traditional monthly SPX (not weeklies), settlement value is technically the SET print at Friday open. The current code approximates with the cash daily close. If the user starts trading AM-settled regulars, add a branch on `t.expiry`'s day-of-month and use the open instead.

## Conventions when editing

- **Don't run `npm audit fix --force`** or auto-bump deps; this codebase is lean by design and `better-sqlite3` builds against the native node version
- **Don't add a build step.** No bundlers, no TypeScript transpilation. Vanilla JS, ES modules.
- **Don't add a frontend framework.** The "modern" parts (Manage page) are vanilla JS + `<script type="module">`. The legacy parts must remain plain inline scripts.
- **When touching the legacy HTML JS:** use Python string-substitution scripts (like the existing patches) to find and replace exact text. The minified single-line blocks make Edit tools brittle.
- **Always test against `/api/metrics/daily/<date>` and `/api/metrics/overall`** after a change that touches trade math.
- **Restart PM2 after server-side edits**: `pm2 restart tradelab-v2`. Frontend-only edits don't need a restart.

## Common tasks

### "My PnL doesn't match my broker"
1. `GET /api/metrics/daily/<date>` — gross and net are both surfaced
2. Compare against broker statement; if broker is net, compare to `net_pnl`
3. Check `GET /api/audit/expired` — open positions get filtered from the dashboard
4. If you're missing trades from a partial-fill scenario, look for trades with the same `entry_dt` and check the executions table

### "Add a new dashboard widget"
- Compute server-side in `server/index.js` (`/api/metrics/...`) and render client-side in the legacy HTML
- For the Manage page, vanilla JS with `<script type="module">` is fine
- Don't introduce React/Vue/etc.

### "Import a new broker's CSV format"
- Add a detector to `csvImport.js` `detectFormat()`
- Write a normalizer that emits the same `{symbol, dt, side, qty, price, commission}` shape that the tape importer already produces
- Reuse the FIFO matcher

### "User wants to delete a date's worth of trades and re-import"
- `DELETE /api/trades` with body `{ids: [...]}` for bulk delete, or one-by-one via `DELETE /api/trades/:id`
- Then `POST /api/trades/import` with the CSV file

## Things that previously broke and how they were fixed

- **Threshold drag was glitchy.** Every `pointermove` called `render()` which rebuilt the SVG, killing pointer capture. Fix: listeners on `document` instead of `svg`; rAF-throttle the render. See `wireScatterBox` in the legacy HTMLs.
- **0 trades after upload.** Broker tape had `Symbol/Side/Fill Price/Time/Net Amount/Commission` columns; importer only knew the IBKR ladder format. Added a third detector + FIFO matcher.
- **Yahoo 429 on every chart load.** Caused by full-browser UA. Switched to `Mozilla/5.0` + curl subprocess. Cached aggressively.
- **`-$812` for a `$97` day.** Dedup bug collapsed FIFO partial-fill trades into one row. Fix: include `exit_dt + exit_price` in dedup key. Restore script `scripts/restore-historical.js` recovered from a known-good HTML snapshot.

## Don't make these mistakes

- Adding `console.log` in `render()` paths — they fire hundreds of times during a scatter drag
- Synchronous DB queries on the request hot path that aren't already there — every existing query is sub-ms because of WAL + prepared statements; new ones should match
- Storing raw broker CSV bytes — the importer extracts what it needs; storing raw is a privacy/space concern
- Calling Yahoo or Massive without the existing cache wrapper — you'll get rate-limited within minutes
